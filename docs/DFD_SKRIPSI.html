1. Context Diagram (DFD Level 0)

Context Diagram menunjukkan gambaran umum sistem dan interaksinya dengan entitas eksternal.

Keterangan Entitas Eksternal:

Entitas Deskripsi
Bidan Petugas medis yang menginput data kelahiran dan mencetak dokumen SKL
Admin Administrator sistem yang mengelola pengguna, dokter, dan memantau aktivitas
Super Admin Administrator dengan akses penuh untuk konfigurasi sistem dan manajemen role
Publik Pihak eksternal yang memverifikasi keaslian dokumen SKL melalui QR Code

Keterangan Proses:

No. Nama Proses Deskripsi
1.0 Autentikasi Login Verifikasi identitas pengguna melalui email dan password
2.0 Manajemen Data Kelahiran CRUD data kelahiran bayi (Create, Read, Update, Delete)
3.0 Generate Dokumen SKL Pembuatan dokumen PDF SKL dengan QR Code verifikasi
4.0 Manajemen Pengguna Pengelolaan data pengguna sistem dan assignment role
5.0 Manajemen Dokter Pengelolaan data dokter penanggung jawab
6.0 Dashboard & Laporan Menampilkan statistik dan ringkasan data sistem
7.0 Audit Trail Pencatatan dan pemantauan aktivitas pengguna
8.0 Verifikasi Dokumen SKL Verifikasi keaslian dokumen SKL melalui QR Code

Keterangan Data Store:

Kode Nama Deskripsi
D1 USERS Menyimpan data pengguna sistem (bidan, admin, super admin)
D2 BIRTH_RECORDS Menyimpan data kelahiran (bayi, ibu, ayah)
D3 DOCTORS Menyimpan data dokter penanggung jawab
D4 SKLS Menyimpan data Surat Keterangan Lahir
D5 ROLES Menyimpan data role dan permission
D6 ACTIVITY_LOG Menyimpan catatan aktivitas pengguna
